# Project Overview

## Name
Parental Bipolar Prodrome Tracker — **Storm Tracker ⚡️**

## Platform
Vercel-hosted React web app (mobile-friendly), to be converted to an iOS app.

## Users
Parents and other caregivers of a teen who is suspected of having bipolar disorder (prodrome phase). Optionally the teen themselves can join their own tracking project, with their entries categorized as self-observations.

## Primary Use Case
Track a teen's behavior privately so caregivers can assemble clinically meaningful observations to share with a clinician for diagnosis. Logging is kept private from the teen to avoid counterproductive dynamics. Supports multiple caregivers so the full care team stays aligned.

## Secondary Use Cases
1. **Clinical reporting** — Present observations in a visually useful format so expensive appointments are as productive as possible.
2. **Pattern analysis & reminders** — Analyze data to surface behavioral patterns, predict likely mood states based on history and known upcoming triggers, and send reminders to support the teen's care during difficult periods.
3. **Menstrual / mood correlation** — Track menstrual cycle alongside mood data and plot both on the same graph to surface hormonal interactions with bipolar mood shifts.

## Key Design Principles
- Speed of entry matters: a tired caregiver should be able to log something useful in under a minute.
- Data integrity: behavior checklists are based on formal DSM diagnostic criteria and are not user-editable (though a custom section can be added).
- Observational transparency: all entries are timestamped and attributed to a specific user so discrepancies between caregiver and self-observations can be surfaced and discussed.
- Privacy: teen-facing and caregiver-facing data are separated by design.
